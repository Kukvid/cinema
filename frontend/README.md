# CinemaBooking Frontend

Красивый и современный frontend для системы управления кинотеатрами, построенный на React и Material-UI.

## Технологии

- **React 18** - UI библиотека
- **React Router v6** - Роутинг
- **Material-UI v5** - Компоненты и дизайн система
- **Axios** - HTTP клиент
- **React Hook Form** - Управление формами
- **date-fns** - Работа с датами
- **Emotion** - CSS-in-JS

## Особенности

### Дизайн и UI/UX
- Темная тема в стиле Netflix с акцентами красного (#e50914) и золотого (#ffd700)
- Градиенты и тени для глубины
- Плавные анимации и hover эффекты
- Адаптивный дизайн (Mobile First)
- Интерактивные компоненты с обратной связью

### Функциональность
- Каталог фильмов с фильтрацией и поиском
- Интерактивная схема выбора мест в зале
- Бронирование билетов с выбором товаров из кинобара
- Применение промокодов и бонусов
- Личный кабинет с историей билетов
- Админ-панель для управления системой

## Установка и запуск

### 1. Установите зависимости
```bash
npm install
```

### 2. Настройте переменные окружения
Скопируйте `.env.example` в `.env`:
```bash
cp .env.example .env
```

Убедитесь, что в `.env` указан правильный URL backend:
```
REACT_APP_API_BASE_URL=http://localhost:8000/api/v1
```

### 3. Запустите приложение
```bash
npm start
```

Приложение откроется по адресу [http://localhost:3000](http://localhost:3000)

### 4. Сборка для продакшена
```bash
npm run build
```

## Основные компоненты

### Header
Адаптивная шапка с:
- Логотипом и навигацией
- Меню пользователя с аватаром
- Кнопкой входа для неавторизованных
- Ссылками на админ-панель для администраторов

### FilmCard
Красивая карточка фильма с:
- Постером с hover эффектом
- Информацией о фильме (жанр, длительность, рейтинг)
- Кнопкой бронирования
- Оверлеем при наведении

### SeatMap
Интерактивная схема зала:
- Визуализация мест (свободно/занято/выбрано)
- Клик для выбора места
- Цветовая кодировка
- Легенда
- Номера рядов

### SessionList
Список сеансов с:
- Группировкой по датам
- Информацией о времени, зале, цене
- Индикатором доступных мест
- Кнопкой выбора сеанса

## UI/UX решения

### Цветовая схема
- **Primary**: #e50914 (Netflix Red) - основные кнопки, акценты
- **Secondary**: #ffd700 (Gold) - второстепенные элементы, иконки
- **Success**: #46d369 (Green) - свободные места, успешные операции
- **Background**: #141414, #1f1f1f - темный фон
- **Text**: #ffffff, #b3b3b3 - текст

### Градиенты
```css
linear-gradient(135deg, #e50914 0%, #b00710 100%) /* Кнопки */
linear-gradient(135deg, #e50914 0%, #ffd700 100%) /* Заголовки */
linear-gradient(135deg, #1f1f1f 0%, #2a2a2a 100%) /* Карточки */
```

### Анимации
- Плавные переходы (0.3s ease)
- Hover эффекты (scale, translateY, shadow)
- Появление контента (fade in)
- Loading индикаторы

### Адаптивность
- Mobile First подход
- Breakpoints: xs, sm, md, lg, xl
- Адаптивная сетка Grid
- Гибкие компоненты

### Обратная связь
- Loading состояния (CircularProgress)
- Сообщения об ошибках (Alert)
- Подтверждения действий (Dialog)
- Валидация форм (React Hook Form)

## API Integration

Все запросы проходят через axios instance с:
- Автоматическим добавлением JWT токена
- Обработкой 401 ошибок (redirect на /login)
- Базовым URL из переменных окружения

## Аутентификация

AuthContext предоставляет:
- `user` - текущий пользователь
- `token` - JWT токен
- `isAuthenticated` - статус авторизации
- `login(email, password)` - вход
- `register(userData)` - регистрация
- `logout()` - выход
- `updateUser(userData)` - обновление профиля

## Защищенные маршруты

PrivateRoute проверяет:
- Авторизацию пользователя
- Роль (для админ-панели)
- Redirect на /login если не авторизован

## Тестовые данные

## Известные ограничения

- QR-коды показываются как заглушка (нужна интеграция с библиотекой)
- Платежи симулируются (нужна интеграция с платежной системой)
- Уведомления отсутствуют (можно добавить Snackbar)
